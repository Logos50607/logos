# 依賴優先原則

## 規則

當策略規劃中出現「由 X 組負責」「去 Y 組註冊」等跨組假設時，必須：

1. **確認目標組是否已有對應專案與介面**。
2. 若無 → 將該基礎建設列為前置待辦，排在依賴它的任務之前。
3. 若有 → 確認介面規格是否滿足需求，不足則提出擴充需求。

## 禁止行為

- 不得在 workflow 中寫出「去 X 組註冊」而 X 組根本沒有註冊機制。
- 不得假設某組「應該」提供某服務而未確認。

## 處理方式

遇到缺口時：

1. 在策略組 AGENT_PLAN.md 中新增前置待辦：為目標組規劃該基礎建設。
2. 在依賴方的 workflow 中標註「前置條件：X 組的 Y 專案就位」。
3. 不阻塞依賴方的其他工作，但該 workflow 的對接步驟暫標為待定。
