# 監察組 (Supervision)

## 核心目標

1. **品質把關**：code review、自動化測試、QA 流程設計與執行結果驗證
2. **權限治理**：審核各組 agent 的受限操作請求（寫檔、執行指令等），記錄稽核日誌，決定允許、拒絕或轉人工

## 專案

```
supervision/
├── permissions/    # 權限治理：受限操作審核、稽核日誌、重執行機制
└── README.md
```

## 設計原則

- 監察組不主動開發功能，只稽核其他組別的輸出與行為
- 權限決定優先依 policy 自動判斷，無法判斷時才 escalate 至人類
- 所有受限操作請求與決定必須寫入稽核日誌，不得略過
- 重執行原始任務時，以最小必要 `--allowedTools` 呼叫，不得過度授權

## 與其他組別的介面

| 方向 | 對象 | 說明 |
|------|------|------|
| 接收 | 所有組 | 受限操作 escalation 請求 |
| 發出 | 聯絡組 | 需人工判斷時，透過聯絡組通知使用者 |
| 發出 | 原請求組 | 審核結果（允許 / 拒絕）與重執行結果 |
