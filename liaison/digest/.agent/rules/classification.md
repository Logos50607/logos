# 訊息分類與評級規則

## 分類設定來源

分類規則優先從 `CLASSIFICATION_CONFIG`（`.env` 指定的設定檔）載入。
**若未指定，使用以下內建預設**。

內建預設只是起點，應根據實際使用情境在設定檔中覆寫或擴充。

## 預設類別（Categories）

| 類別 | 說明 | 範例 |
|------|------|------|
| `task` | 需要執行某件事 | 「幫我查一下…」、「記得…」 |
| `question` | 提問或需要回覆 | 「這個怎麼用？」 |
| `info` | 純資訊，不需行動 | 新聞分享、公告 |
| `social` | 社交對話 | 問候、閒聊 |
| `alert` | 警示或異常通知 | 系統告警、緊急事項 |
| `unknown` | 無法分類 | 分類信心低於閾值時 |

一則訊息可同時屬於多個類別，但必須指定一個**主類別**。

## 優先級（Priority）

| 級別 | 觸發條件 | 行動 |
|------|----------|------|
| `critical` | `alert` 類、明確要求立即回應 | 立即呈報 |
| `high` | `task` 或 `question` 類 | 下次定期摘要優先列出 |
| `normal` | `info`、`social` 類 | 列入定期摘要 |
| `low` | 重複資訊、已知公告 | 可省略或摺疊 |

## 評級規則

1. **不得對同一訊息評級兩次**：以訊息 ID 為 key，評級結果應快取
2. **不確定時降級**：無法判斷優先級時，降為 `normal`，不要猜測為 `critical`
3. **設定檔優先**：若設定檔定義了關鍵字或規則，完全依照設定，內建預設不介入

## 輸出格式

每則分類後的訊息應包含：

```json
{
  "msg_id": "...",
  "channel": "line-personal",
  "timestamp": "ISO8601",
  "sender": "...",
  "text": "...",
  "category": "task",
  "priority": "high",
  "has_media": false
}
```
