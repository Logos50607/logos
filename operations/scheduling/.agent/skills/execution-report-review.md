# 執行報告審閱

## 用途

對指定時間範圍內的執行報告進行彙整分析，產出趨勢報告。供營運組自我監管，亦可作為聯絡組「回報統整」的深度分析輸入。

## 輸入

- **時間範圍**：如「最近 7 天」、「2026-03-01 至 2026-03-26」
- **篩選條件**（可選）：特定任務名稱、特定 owner_group、特定 status

## 分析邏輯

### 1. 彙整統計

- 各任務的執行次數、成功次數、失敗次數、成功率。
- 各任務的平均耗時、最長耗時、最短耗時。
- 全系統的總任務數、總執行次數、整體成功率。
- 各 owner_group 的任務執行統計。

### 2. 趨勢識別

- **成功率下降**：與前一週期（相同天數）比較，成功率降幅超過 10% 的任務。
- **執行時間增長**：平均耗時較前一週期增長超過 20% 的任務。
- **任務變動**：新加入的任務、被停用的任務、被移除的任務。
- **重試頻率**：需要重試才能成功的任務（平均 attempt > 1）。

### 3. 異常標記

- **從未成功的新任務**：註冊後每次執行皆失敗。
- **突然失敗的穩定任務**：過去持續成功但近期開始失敗。
- **報告缺失**：registry 有執行紀錄但找不到對應報告。
- **降級執行增多**：`status=partial` 次數上升的任務。

## 輸出

趨勢報告（Markdown），結構如下：

```markdown
## 排程執行趨勢報告

**分析期間**：YYYY-MM-DD 至 YYYY-MM-DD
**任務總數**：N
**總執行次數**：N
**整體成功率**：N%

### 各任務統計

| 任務 | 執行次數 | 成功率 | 平均耗時 | 趨勢 |
|------|---------|--------|---------|------|
| ... | ... | ... | ... | ↑/↓/→ |

### 趨勢警示

- 成功率下降：（列表）
- 耗時增長：（列表）
- 重試頻繁：（列表）

### 異常標記

- 從未成功：（列表）
- 突然失敗：（列表）
- 報告缺失：（列表）

### 建議

1. ...
2. ...
```

## 觸發建議

- 每週一次，附加於週報中。
- 手動觸發：需要向人類或其他組別報告排程運行狀況時。
