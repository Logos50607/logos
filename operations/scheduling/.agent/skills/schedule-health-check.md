# 排程健康檢查

## 用途

對排程系統的整體狀態進行健康檢查，產出健康報告。可被 `daily-dispatch` 定期呼叫（建議每週一次），亦可手動觸發用於問題排查。

## 檢查維度

### 1. Registry 一致性

- registry 中的每個任務是否都能在來源專案找到對應 manifest（manifest 未被刪除）。
- 是否有 manifest 存在於專案中但未被收集至 registry（孤立 manifest）。
- `registry/invalid.md` 中是否有長期未修正的不合規 manifest。

### 2. 執行狀態

- **過期未執行**：距上次執行超過 2 倍排程間隔的任務。
- **連續失敗**：最近 3 次以上執行結果為 failure 的任務（含已自動停用的）。
- **長期停用**：`enabled=false` 超過 30 天的任務，可能需要清理或重新啟用。
- **從未執行**：已註冊但從未產出任何執行報告的任務。

### 3. 排程密度

- 統計各時段的排程任務數量，識別過度集中的時段。
- 是否有任務的平均執行時間超過排程間隔（任務來不及完成就要再次觸發）。
- 同一 `working_dir` 是否有多個任務可能同時執行而產生衝突。

### 4. 報告完整性

- 每次執行是否都有對應的 `EXECUTION_REPORT`（registry 有紀錄但 reports/ 無報告）。
- 報告的必填欄位是否完整。
- daily-summary 是否有缺漏的日期（排程系統可能當日未運作）。

## 輸出

健康報告（Markdown），結構如下：

```markdown
## 排程系統健康報告

**檢查時間**：YYYY-MM-DD HH:MM
**整體狀態**：健康 / 需關注 / 有異常

### Registry 一致性
- 有效任務數：N
- 孤立 manifest：N（列表）
- 不合規 manifest：N

### 執行狀態
- 過期未執行：N（列表）
- 連續失敗：N（列表）
- 長期停用：N（列表）

### 排程密度
- 高峰時段：HH:MM（N 個任務）
- 執行時間超過間隔：N（列表）

### 報告完整性
- 缺漏報告：N
- 缺漏日報日期：N

### 建議處置
1. ...
2. ...
```

## 觸發建議

- 由 `daily-dispatch` 每週一附加於日報中。
- 手動觸發：當懷疑排程系統有異常時。
