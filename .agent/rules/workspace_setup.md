# 工作區結構與清單規範 (Workspace Setup & Lists)

本規則旨在確保工作區結構化，並透過清單管理 AI 與使用者的互動。

## 1. 核心結構

每當開啟新資料夾或工作區時，必須確保存在：

- `.agent/rules/`：存放專案規範。
- `.agent/skills/`：存放專案輔助技能。
- `.agent/workflows/`：存放 slash commands 及自動化流程。

## 2. 管理清單 (TODOLIST & ASKHUMAN)

在工作區根目錄必須始終維護：

- **`TODOLIST.md`**：記錄 AI 計畫執行的事項。
- **`ASKHUMAN.md`**：記錄需請示使用者的項目、衝突或未驗收的任務。
- **格式規範**：
  - 使用條列表 (`- [ ]`, `- [x]`)。
  - 標號後加上日期 (格式：`YYYY-MM-DD`)。
  - **最新日期優先**：新加入的項目應置於文件最上方。

## 3. README 優先原則

- **主動讀取**：對話開始、分析功能或進入子目錄時，優先讀取該目錄下的
  `README.md`。
- **維護職責**：確保 `README.md`
  清楚說明目錄結構、程式碼邏輯及使用方式，以便後續開發。
