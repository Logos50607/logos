# TODOLIST Management Skill

此 Skill 用於規範與管理專案根目錄下的 `TODOLIST.md`。 **核心目的**:
此文件**僅供使用者 (USER) 紀錄**需裁斷或執行的事項。AI 的工作進度應體現在 Commit
訊息中。

## 格式規範 (Format Rules)

1. **條列式撰寫**: 所有項目必須以 Markdown 條列式 (List) 撰寫。
2. **Checkbox**: 在每項條列標號前面，必須包含一個 Checkbox (`- [ ]` 或
   `- [x]`)。
3. **日期紀錄**: 在條列標號（或
   Checkbox）之後、具體內容之前，必須加上該項目的**加入日期**，格式為
   `YYYY-MM-DD`（例如：`2026-03-06`）。
4. **排序規範**: **新加入（日期較晚）的項目必須放在文件的最上方**。
5. **內容分組**:
   （可選）可以根據「尚未裁斷的衝突」、「尚未完成項」、「未來優化」進行二級標題分組。

## 執行步驟

- 當需要更新 TODO 時，主動讀取此格式規範。
- 確保新加入的項目都帶有當前日期。
- 已完成的項目標註為 `[x]`。
